Highlight #1 Version Control

This allows it so that files can't be overwritten because there is a common repository that holds the latest files. It also allows people to work on the same files without any conflict at the same time, plus it allows you to revert back to an older version of the file. Even without a team it will allow you to back up your files so you don't lose your project.

Highlight #2 GIT

GIT is an free open source control system that is ment to handle everything in a fast and efficent manner, reguardless of if it is a large or a small project. GIT will also prevent unintentioanl overwrites of a file share repository.

Highlight #3 Remote Repository

When a repository is created the developer can record and change its history and track its status so a remote repository is a repository that resides on a remote server that is being shared among a number of team members.

Highlight #4 Local Repository

Unlike the Remote repository that uses a remote server the local repository resides on a local machine from an individual. You can create this by creating a brand new repository or you can close an existing remote repository on your local machine. This seems to be very beneficial to people who are working on an individual project and doesn't need to deal with having to log into a specific server.

Highlight #5 Why use GIT?

We would want to use GIT because it stores a copy of the repository data on their on personal computer, and since everything is local you can work from your own computer. GIT runs very fast and will prevent unintentional overwrites. I personally would want to use this because I like the idea of everything being on my own local computer and not having to deal with other servers.
